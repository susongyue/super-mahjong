## 角色详情展开动画（game.html + battle_round.html）

重做角色详情界面，点击卡片后从卡片位置平滑过渡展开为居中详情面板，关闭时反向收缩回原卡片位置。

**实现方式**：GSAP 手动 FLIP 动画

1. **事件绑定**：点击时传递被点击的卡片 DOM 元素给 `openFullscreen(idx, cardEl)`
2. **openFullscreen 展开**（~0.45s）：
   - 获取卡片 `getBoundingClientRect()`
   - `gsap.set` overlay 初始位置/大小与卡片完全重合
   - 初始背景色设为卡片背景色（防止颜色跳变）
   - 内容元素 opacity 初始为 0
   - `gsap.to` 动画到居中目标位置（`top: 5vh`, `left: 居中`, `width: min(900px, 92vw)`）
   - 背景色同步过渡到 `var(--canvas)`
   - 内容元素在动画后半段 stagger 淡入
3. **closeFullscreen 收缩**（~0.35s）：
   - 内容先淡出（0.15s）
   - overlay 收缩回原卡片位置/大小
   - 完成后 `clearProps: 'all'` 清理 inline 样式并隐藏
4. **防重复点击**：`fsAnimating` 标志位阻止动画过程中再次触发

涉及文件: `public/game.html`, `public/battle_round.html`

## 背景模糊分离到独立 backdrop 层

之前 `backdrop-filter` 错误地加在 `.char-fullscreen-overlay` 上，导致模糊区域是 overlay 自身透过半透明背景的效果，而非整个页面背景。

修复方案：
1. **新增 `.char-fullscreen-backdrop`** — 独立的全屏固定层 (`z-index: 299`)，带 `backdrop-filter: blur(20px) saturate(180%)` + 半透明黑色遮罩，负责模糊整个页面背景
2. **`.char-fullscreen-overlay`** 改为实色背景 `var(--canvas)`，去掉 backdrop-filter，作为详情面板坐在模糊背景之上 (`z-index: 300`)
3. **GSAP 动画分离**：backdrop 和 overlay 并行展开/收缩（同 duration、同 ease），blur 从 0 逐渐过渡到 20px，面板背景从卡片颜色过渡到 `var(--canvas)`

## style.css skill-tag 用 color-mix() 替代三属性方案

将 `.skill-tag.limit/.passive/.innate/.extra` 四个变体从三变量方案（`background: var(--xxx-bg); color: var(--xxx); border-color: var(--xxx-border)`）改为 `color-mix()` 函数派生：

- `--_c: var(--xxx)` 作为基色
- `color: var(--_c)` — 文字色
- `background: color-mix(in srgb, var(--_c) 12%, transparent)` — 背景 12% 不透明度
- `border-color: color-mix(in srgb, var(--_c) 30%, transparent)` — 边框 30% 不透明度

这样每个变体只需一个基色变量，bg/border 由 `color-mix()` 自动计算。
